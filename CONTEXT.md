# Bilibili 硬核会员答题助手

本上下文描述一个运行在手机上的答题辅助软件，用于帮助用户理解并回答 bilibili 硬核会员相关的圈子知识测试题。

## 语言

**知识测试**:
一种面向社区或兴趣圈子的问答测试，用来判断用户对特定圈子、内容领域或 bilibili 主题的了解程度。
_避免_: 正式考试、监考考试

**单选题**:
MVP 支持的唯一题型，由题干和若干可选答案组成，用户根据答案提示自行选择一个答案。
_避免_: 多选题、判断题、填空题、问答题

**答案提示**:
答题助手读取题目并检索相关知识后展示给用户的推荐答案、简短依据和可信程度。
_避免_: 自动答题、作弊自动化

**顶部提示条**:
默认显示在屏幕正上方的紧凑答案提示，尽量避免遮挡题干和选项。
_避免_: 全屏弹窗、遮挡答题区

**展开提示**:
用户主动展开顶部提示条后看到的更多依据、来源或错误信息。
_避免_: 默认长文本、聊天窗口

**答案结果**:
后端根据题目文本返回给手机端的结构化内容，用于生成答案提示。
_避免_: 大模型原文、自由文本回复

**低置信答案**:
依据不足但仍可能对用户有帮助的候选答案，展示时必须明确标注可信程度较低。
_避免_: 确定答案、失败结果

**悬浮控制**:
用户停留在其他应用内时，用来启动或停止读屏的一小块屏幕控件。
_避免_: 插件、浏览器扩展

**悬浮状态**:
悬浮控制在未启动、待识别、识别中、展示结果和失败之间切换时呈现给用户的状态。
_避免_: 后台状态、内部任务状态

**按需授权**:
用户触发需要系统权限的操作时，答题助手检查权限状态，并在缺少权限时阻塞当前操作等待用户授权。
_避免_: 首次启动权限引导、提前授权所有权限

**权限兜底**:
用户拒绝、取消或系统不支持某项权限时，答题助手给出可理解的失败状态或替代路径，而不是继续执行不可用操作。
_避免_: 静默失败、无限重试

**单次识别**:
用户主动触发后，答题助手只读取当前屏幕画面并尝试识别当前题目的一次性操作。
_避免_: 持续监听、自动扫题

**端侧识别**:
答题助手在用户手机本地完成屏幕文字识别，只将识别出的题目文本用于后续检索和答案生成。
_避免_: 云端截图识别、上传完整屏幕

**屏幕画面识别**:
答题助手仅根据用户设备屏幕上可见的文字识别题目，不依赖 bilibili 账号、Cookie、接口或页面结构。
_避免_: 页面接口读取、登录态接入、DOM 注入

**读屏会话**:
用户授权并启动的一段屏幕捕获过程，答题助手在会话有效期间允许用户手动触发单次识别。
_避免_: 永久截屏权限、无限期后台录屏

**会话失效**:
读屏会话因用户结束、系统回收授权、应用进程中断或捕获失败而不可继续使用的状态。
_避免_: 自动恢复、静默继续读屏

**题目文本**:
端侧识别从屏幕中提取出的题干、选项和相关上下文文字，是手机端请求后端答案提示的主要输入。
_避免_: 屏幕截图、页面图像

**原始题目文本**:
端侧识别直接得到的未经完整结构化处理的题目文字，是请求后端答案提示时的兜底输入。
_避免_: 解析失败、空请求

**题目结构**:
从原始题目文本中尽力拆分出的题干、选项和其他上下文信息。
_避免_: 必填格式、固定模板

**知识库**:
答题助手用于匹配题目、选项、解释和来源资料的结构化或半结构化资料集合。
_避免_: 题库脚本、答案表

**知识库条目**:
知识库中描述一道题及其答案依据的最小内容单元，包含题干、选项、答案、解释、来源和标签。
_避免_: 单条答案、文本片段

**JSON 知识库文件**:
人工整理并导入后端知识库的 JSON 文件，是 MVP 维护知识库条目的默认载体。
_避免_: 自动爬取数据、管理后台录入

**后端知识库**:
由后端服务维护和查询的知识库，手机端通过识别出的题目文本请求答案提示。
_避免_: App 内置完整题库、本地答案包

**匿名质量日志**:
后端为改进答案质量和排查错误而记录的非用户侧历史数据，可以包含完整题目文本，但不包含截图或用户身份。
_避免_: 用户答题历史、个人记录

**混合检索**:
答题助手先查询知识库，命中不足时再使用联网搜索和大模型生成答案提示的答案获取方式。
_避免_: 纯大模型猜题、只搜网页

**搜索来源白名单**:
后端进行可控联网搜索时允许使用的公开信息来源范围，用于控制答案依据的质量和噪声。
_避免_: 私有接口依赖
